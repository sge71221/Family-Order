## 一、产品概述

### 1.1 产品定位

**自用家庭点菜微信小程序**——专为家庭内部使用设计的轻量化点菜、记账、采购一体化工具。无对外营业属性，无需营业执照，仅家人之间共享菜单、协作点餐、记录开销、管理采购。

### 1.2 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| 家庭管理员（主人） | 负责菜品录入、采购管理的家庭成员（通常是做饭的人） | 菜品管理、采购规划、成本核算 |
| 普通家庭成员 | 爸爸/妈妈/孩子/长辈等日常点餐者 | 快速点餐、表达口味偏好、查看今日菜单 |
| 老年长辈 | 操作习惯偏传统、视力可能受限的家庭成员 | 大字体、简单操作、少步骤完成点餐 |
| 儿童成员 | 10岁以上可独立操作手机的孩子 | 界面直观、趣味性、选自己想吃的 |

### 1.3 核心价值

1. **告别"今天吃什么"的纠结**——共享菜单一目了然，全家人一起选
2. **采购不再遗漏**——自动提取食材清单，买菜一步到位
3. **家庭开支透明**——每顿饭花了多少钱，月度统计心中有数
4. **健康饮食适配**——忌口过敏自动拦截，老人控糖小孩营养均衡
5. **操作零门槛**——老人小孩都能独立完成点餐，无需学习成本

### 1.4 产品目标

| 目标 | 衡量指标 |
|------|----------|
| 降低家庭日常饮食决策成本 | 单次点餐从提出到确认 ≤ 3 分钟 |
| 减少食材采购遗漏率 | 采购清单与实际需求匹配度 ≥ 95% |
| 提升家庭饮食开支管理意识 | 月度饮食开销统计查看率 ≥ 80% |

## 二、用户故事

### 2.1 菜品管理模块

| ID | 用户故事 |
|----|----------|
| US-M01 | As 家庭管理员，I want 新增菜品并上传图片，so that 家人看到菜品图就能直观选择 |
| US-M02 | As 家庭管理员，I want 给菜品设置分类和标签（如"快手菜""儿童餐""硬菜"），so that 家人能快速筛选找到想吃的类型 |
| US-M03 | As 家庭管理员，I want 录入菜品食材配料及难易度、烹饪时长，so that 能评估做菜时间和采购需求 |
| US-M04 | As 家庭管理员，I want 批量导入已有菜品列表，so that 不需要逐条手动录入家常菜 |
| US-M05 | As 家庭管理员，I want 对菜品进行上下架操作，so that 季节性或临时不做菜品可以隐藏不干扰家人选择 |
| US-M06 | As 普通家庭成员，I want 搜索菜品名称或标签，so that 能快速找到特定的菜品 |

## 四、功能规格详述

### 4.1 菜品管理模块

#### 4.1.1 菜品新增

- **入口**：底部导航"菜单"页 → 页面右上角 "+" 按钮（仅管理员可见）
- **输入字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 菜品名称 | 文本（≤20字） | 是 | 菜品的显示名称 |
| 分类 | 单选枚举 | 是 | 家常菜/荤菜/素菜/汤品/主食/小吃/甜品/速食 |
| 菜品图片 | 图片（1-3张） | 否 | 支持拍照/从相册选择，首张图为主图 |
| 菜品备注 | 多行文本（≤200字） | 否 | 辣度、做法要点、特色说明 |
| 标签 | 多选标签 | 否 | 从已有标签池选取，或新建自定义标签 |
| 食材配料 | 关联列表 | 否 | 关联食材库中的食材及用量 |
| 难易度 | 单选（简单/中等/复杂） | 否 | 标注做菜难度 |
| 烹饪时长 | 数字（分钟） | 否 | 预估烹饪时长 |
| 上架状态 | 开关 | 否 | 默认上架，可设为下架 |

- **交互流程**：
  1. 管理员点击"+" → 进入菜品新增表单页
  2. 依次填写各字段，图片可跳过后续补充
  3. 食材配料：点击"添加食材"→ 弹出食材库搜索列表 → 选择食材并输入用量（如"土豆 2个"）
  4. 标签：点击"添加标签"→ 弹出已有标签列表供勾选，底部有"+自定义标签"入口
  5. 点击"保存"→ 数据写入云数据库 + 更新本地缓存 → 跳转回菜单列表页
  6. 如果图片未上传，菜品卡片显示默认占位图（碗碟图标）

- **边界情况**：
  - 菜品名称重复时提示"已有同名菜品，是否合并信息？"
  - 图片上传失败（网络问题）时自动降级为本地暂存，下次联网时自动上传
  - 食材库中无所需食材时，支持在配料步骤中快速新建食材（名称+单位+单价）

- **数据结构**：见第六章 `Dish` 实体

#### 4.1.2 菜品分类浏览

- **入口**：底部导航"菜单"页
- **交互流程**：
  1. 默认展示"全部"上架菜品卡片网格（2列），每张卡片：主图 + 菜品名 + 标签摘要
  2. 页面顶部横向滑动的分类 Tab：全部 | 家常菜 | 荤菜 | 素菜 | 汤品 | 主食 | 小吃 | 甜品 | 速食
  3. 选择分类后，下方卡片列表即时筛选刷新
  4. 卡片点击 → 进入菜品详情页（图片轮播 + 名称 + 备注 + 食材配料 + 标签 + 难易度/时长）
  5. 详情页右上角"编辑"按钮（仅管理员可见）→ 进入菜品编辑页
  6. 详情页右下角"收藏"按钮（所有成员可用）→ 收藏/取消收藏

- **排序规则**：上架菜品优先，按创建时间倒序（最新在上）；收藏菜品可单独筛选

#### 4.1.3 菜品搜索

- **入口**：菜单页顶部搜索框
- **交互流程**：
  1. 点击搜索框 → 输入关键词 → 实时匹配菜品名称和标签
  2. 搜索结果即时展示卡片列表
  3. 无结果时显示"未找到相关菜品，试试其他关键词"
  4. 搜索历史本地缓存（最近10条），下拉搜索框时展示搜索历史

#### 4.1.4 菜品上下架

- **交互流程**：
  1. 管理员在菜品编辑页或菜品详情页可切换"上架/下架"开关
  2. 下架菜品在普通成员视角完全不可见
  3. 管理员在菜单页可通过筛选"已下架"查看和管理下架菜品
  4. 下架时弹出确认提示："确认下架【菜品名】？家人将无法看到此菜品"
  5. 重新上架即时生效，无需二次审核

#### 4.1.5 标签自定义

- **交互流程**：
  1. 管理员在菜品新增/编辑页点击"自定义标签"→ 输入标签名（≤6字）→ 保存
  2. 自定义标签加入全局标签池，所有成员在选择分类/标签筛选时可使用
  3. 标签删除：管理员可删除自定义标签（系统预设标签不可删），删除后关联菜品的该标签自动移除
  4. 预设标签池：清淡、重辣、快手菜（≤30分钟）、硬菜（耗时复杂）、儿童餐、养生汤、下饭菜、节日菜

#### 4.1.6 批量导入菜品（P2）

- **交互流程**：
  1. 管理员在菜单页右上角菜单 → "批量导入"→ 提供模板下载（CSV格式）
  2. 填写模板后上传 → 解析并逐条创建菜品
  3. 解析失败行提示具体原因（如缺少必填字段）
  4. 批量导入的菜品默认无图片，管理员后续可逐个补充

### 4.2 家庭点餐下单模块

#### 4.2.1 点餐购物车

- **入口**：底部导航"点餐"页
- **交互流程**：
  1. 进入点餐页 → 显示今日上架菜品卡片网格（与菜单页布局一致，但增加操作按钮）
  2. 每张卡片右下角有"+"按钮 → 点击后菜品加入购物车，份数默认1
  3. 已加入购物车的菜品，卡片右下角显示"份数 +/-"控件
  4. 页面底部悬浮购物车摘要条：显示已选菜品数 + 总份数 + "查看购物车"按钮
  5. 点击"查看购物车"→ 进入购物车详情页

- **购物车详情页**：
  - 每条已选菜品行：菜品缩略图 + 名称 + 份数（+-按钮）+ 单行备注输入框
  - 备注输入：点击备注框 → 弹出快捷备注选项（不吃葱/不吃蒜/少盐/少油/多辣/不要味精）+ 自定义输入
  - 底部操作区："一键清空"按钮（红色） | "提交订单"按钮（绿色大按钮）

- **边界情况**：
  - 购物车为空时提示"还没选菜哦，去看看今日菜单吧"
  - 同一菜品添加多份时合并显示（份数叠加），备注分别记录
  - 点餐时段：管理员可设置"今日点餐开放时间"（如9:00-17:00），超出时段提示"今日点餐已截止"

#### 4.2.2 点餐提交与汇总

- **交互流程**：
  1. 成员点击"提交订单"→ 弹出确认框："确认提交今日点餐？共N道菜M份"
  2. 确认后 → 订单写入云数据库 + 本地缓存 → 提交成功提示
  3. 提交后购物车自动清空，页面跳转到"今日点餐汇总"页
  4. 已提交订单的成员在点餐页顶部看到"已提交今日订单"状态栏，可点击"修改订单"追加或减少

- **今日点餐汇总页**：
  - 按菜品维度聚合：显示每道菜品的总份数 + 哪些人点了 + 各人备注
  - 格式示例：
    ```
    🍲 番茄炒蛋 × 3份
      妈妈(1份) · 孩子(2份, 少盐)
    🥩 红烧排骨 × 2份  
      爸爸(1份) · 长辈(1份, 不吃辣)
    ```
  - 页面顶部统计：今日共 N 人点餐、M 道菜、K 份
  - 管理员专属操作："标记开始做饭"→ 订单状态变为"制作中"，锁定不再接受修改

- **数据同步**：
  - 多人同时点餐时，云数据库实时监听（watch）订单变化
  - 每位成员提交后，汇总页实时刷新（1-3秒延迟）
  - 如果网络不通，本地先暂存订单，联网后自动同步到云端

#### 4.2.3 历史点餐记录

- **入口**：点餐页顶部"历史记录"Tab
- **交互流程**：
  1. 展示最近30天点餐记录列表，每条：日期 + 点餐人数 + 菜品数
  2. 支持筛选：按日期范围 / 按点餐人
  3. 点击某条记录 → 查看当日详细点餐清单（同今日汇总格式）
  4. 支持删除历史记录（管理员权限）

### 4.3 食材采购与库存模块

#### 4.3.1 自动提取采购清单

- **触发时机**：订单提交后 或 管理员手动触发
- **交互流程**：
  1. 进入"采购"页 → 自动显示基于今日订单提取的食材清单
  2. 清单按菜品维度展开：每道菜下方列出所需食材及用量
  3. 相同食材自动合并总量（如番茄炒蛋需要2个番茄 + 番茄汤需要3个番茄 → 合计5个番茄）
  4. 已有库存食材标注库存量，显示"需采购量 = 总需求量 - 库存量"
  5. 清单中每条食材行：食材名 + 需采购量 + 勾选框 + 单价（如有录入）

- **边界情况**：
  - 菜品未关联食材配料时，采购清单中该菜品显示"未录入食材，请手动补充"
  - 用量单位不一致时（如1个土豆 vs 200g土豆），以管理员在食材库中定义的基准单位统一换算

#### 4.3.2 采购清单勾选与导出

- **交互流程**：
  1. 采购页清单中，管理员逐条勾选需要采购的食材
  2. 点击"导出清单"→ 生成纯文本格式，示例：
    ```
    🛒 今日采购清单（2026-07-02）
    1. 番茄 × 5个
    2. 排骨 × 500g
    3. 鸡蛋 × 6个
    4. 盐（家里还有，不需要）
    ```
  3. 文本内容自动复制到剪贴板 + 提供"转发到微信"按钮
  4. 转发到微信：调用微信小程序分享 API，以卡片消息形式发送到微信群

#### 4.3.3 食材库存登记

- **入口**：采购页 → "库存管理"Tab
- **交互流程**：
  1. 展示食材库存列表：食材名 + 当前数量 + 保质期（如有）+ 单价 + 类型（常备/临时）
  2. 点击某食材 → 进入编辑页，可修改数量、保质期、单价
  3. 新增食材：点击"+" → 输入食材名称、单位（个/斤/包/瓶等）、数量、单价、保质期、类型
  4. 食材库与菜品配料共享同一食材数据池

- **食材消耗自动扣减**：
  - 订单标记"已做饭"后，系统自动扣减订单所需食材的库存数量
  - 扣减前弹出确认："将自动扣减以下食材库存：番茄-5个，排骨-500g，确认？"
  - 库存不足时标红提示"库存不足：番茄当前2个，需5个，差3个"

#### 4.3.4 食材保质期预警

- **规则**：
  - 保质期剩余 ≤ 3天：黄色标记"即将过期"
  - 保质期已过：红色标记"已过期"
  - 进入采购页时，顶部预警栏展示所有临近过期食材列表
  - 点击预警食材 → 建议优先使用的菜品列表（关联含有该食材的菜品）

#### 4.3.5 食材单价与成本计算

- **交互流程**：
  1. 管理员在食材库中录入每种食材单价
  2. 系统自动计算：每道菜食材成本 = Σ(食材用量 × 单价)
  3. 单次订单总花费 = Σ(各菜品食材成本 × 份数)
  4. 成本数据展示在订单汇总页和记账统计页

- **方式2：家庭码加入**
  - 已有家庭成员分享家庭码 → 新成员输入家庭码 → 选择身份 → 加入
- **登录后行为**：
  - 自动加载家庭菜品数据到本地缓存
  - 后续打开小程序自动登录（openid + 本地token缓存）

- **技术实现**：微信云开发实时数据推送（watch API）
- **交互效果**：

  - 如果推送失败，页面提供手动刷新按钮

#### 4.6.5 离线缓存

- **缓存策略**：

  - 食材库：本地缓存全量食材数据
  - 个人忌口设置：本地缓存
  - 购物车：本地暂存，联网后同步
  - 订单提交：本地队列暂存，联网后依次上传

- **离线限制**：
  - 离线可浏览菜单、添加购物车、设置忌口
  - 离线不可提交订单（需联网确认）、不可查看实时汇总、不可修改菜品
  - 恢复网络后自动执行同步队列，并在页面顶部提示"已同步N条数据"

## 五、UI 设计规格

### 5.1 页面结构总览

```
底部导航栏（固定）
┌─────────┬─────────┬─────────┬─────────┬─────────┐
│  🍽 菜单 │  📋 点餐 │  🛒 采购 │  📊 统计 │  👤 我的 │
└─────────┴─────────┴─────────┴─────────┴─────────┘

菜单页：分类Tab + 菜品卡片网格 + 搜索框
点餐页：菜品卡片(带+) + 购物车摘要条 + 今日汇总
采购页：采购清单Tab + 库存管理Tab + 保质期预警栏
统计页：月度概览 + 消费图表 + 爱吃排行Tab
我的页：个人信息 + 忌口设置 + 家庭管理(管理员) + 数据备份
```

### 5.2 导航体系

- **主导航**：底部固定五栏，图标+文字，当前页高亮暖橙色
- **次级导航**：
  - 菜单页：顶部横向分类Tab
  - 点餐页：今日点餐 / 历史记录 Tab
  - 采购页：采购清单 / 库存管理 Tab
  - 统计页：月度概览 / 爱吃排行 Tab
- **页面跳转**：
  - 卡片点击 → 菜品详情页（从底部滑入）
  - "+"操作 → 表单页（从右侧滑入）
  - 返回：左上角返回箭头 或 下拉返回手势

### 5.3 交互规范

| 规范项 | 定义 |
|--------|------|
| 点击反馈 | 所有可点击元素有0.1s缩放动画（95%→100%） |
| 加载状态 | 骨架屏占位，不超过3秒白屏 |
| 弹窗样式 | 底部抽屉式弹窗（从底部滑入），避免居中断操作流 |
| 确认操作 | 删除/清空/下架等破坏性操作必须二次确认弹窗 |
| 成功提示 | 页面顶部3秒绿色Toast，自动消失 |

## 七、非功能需求

### 7.1 性能需求

| 项目 | 指标 |
|------|------|
| 小程序启动时间 | ≤ 2秒（冷启动）/ ≤ 1秒（热启动） |
| 菜品列表渲染 | ≤ 1秒展示首批20条 |
| 图片加载 | 单张 ≤ 2秒，失败降级 ≤ 0.5秒 |
| 订单提交响应 | ≤ 3秒完成提交并返回 |
| 实时同步延迟 | ≤ 3秒跨成员数据刷新 |
| 本地缓存命中 | 离线场景下核心功能 ≤ 0.5秒响应 |
| 云数据库查询 | 单次查询 ≤ 500ms |

### 7.2 安全需求

| 项目 | 规范 |
|------|------|
| 数据隔离 | 家庭间数据严格隔离，familyId 为所有查询的必带过滤条件 |
| 权限校验 | 管理员操作（菜品编辑/删除/上下架/成员管理）必须校验 role=admin |
| 输入校验 | 所有用户输入字段长度/格式前端校验 + 云函数二次校验 |
| 图片安全 | 上传图片经微信云存储处理，限制 ≤ 2MB/张 |
| 登录安全 | openid 为唯一身份标识，不暴露给其他成员 |
| 数据备份 | 云数据库自动备份，管理员可导出 JSON 备份文件 |

### 7.3 兼容性需求

| 项目 | 规范 |
|------|------|
| 微信版本 | ≥ 7.0.0（支持云开发） |
| iOS 版本 | ≥ iOS 10 |
| Android 版本 | ≥ Android 5.0 |
| 手机屏幕 | 适配 375px-428px 宽度（iPhone SE ~ iPhone Pro Max） |
| 大字体模式 | 最大放大至标准字体的 1.5倍 |
| 深色模式 | 跟随系统设置 + 手动切换 |

### 7.4 离线能力需求

| 项目 | 规范 |
|------|------|
| 离线可用功能 | 浏览菜单、添加购物车、设置忌口、查看本地缓存的历史记录 |
| 离线不可用功能 | 提交订单、查看实时汇总、编辑菜品、管理成员 |
| 网络恢复同步 | 自动执行本地暂存队列，1分钟内完成全部同步 |
| 离线提示 | 页面顶部显示"当前离线，部分功能不可用"提示条 |
| 缓存容量 | 本地缓存上限 5MB，超出时优先淘汰最久未更新数据 |

### 7.5 可维护性需求

| 项目 | 规范 |
|------|------|
